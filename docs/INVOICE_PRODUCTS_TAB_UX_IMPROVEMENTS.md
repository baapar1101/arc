# پیشنهادات بهبود رابط کاربری تب کالاها و خدمات در فرم فاکتور

## 📋 خلاصه اجرایی

این سند پیشنهادات بهبود رابط کاربری بخش افزودن و ویرایش ردیف‌های کالا/خدمت در فرم فاکتور را ارائه می‌دهد. هدف اصلی: **ثبت سریع‌تر فاکتور** با تجربه کاربری بهتر در دسکتاپ و موبایل.

---

## 🔍 وضعیت فعلی

### ساختار فعلی
- **سطر اول**: شماره، کالا/خدمت، تعداد+واحد، انبار (اختیاری)، قیمت واحد، مبلغ کل، حذف
- **سطر دوم**: شرح، تخفیف، مالیات
- **سطر سوم**: انتخاب instance های یونیک (فقط برای کالاهای یونیک)

### مشکلات شناسایی شده

#### 1. **اندازه المان‌ها**
- ❌ ارتفاع فیلدها: `36px` (خیلی کوچک برای لمس)
- ❌ اندازه‌های ناهماهنگ (بعضی 36px، بعضی 56px)
- ❌ فونت‌ها و آیکون‌ها کوچک هستند

#### 2. **Responsive Design**
- ❌ استفاده از `Row` با `Flexible` که در موبایل فشرده می‌شود
- ❌ عدم استفاده از `ResponsiveHelper` موجود در پروژه
- ❌ در موبایل المان‌ها روی هم می‌افتند یا خیلی کوچک می‌شوند

#### 3. **چیدمان و مرتب‌سازی**
- ❌ امکان drag & drop برای تغییر ترتیب ردیف‌ها وجود ندارد
- ❌ دکمه‌های جابجایی (↑↓) وجود ندارد
- ❌ شماره ردیف‌ها فقط نمایشی است

#### 4. **سرعت ثبت**
- ❌ نیاز به کلیک/لمس زیاد برای پر کردن یک ردیف
- ❌ فیلدها کوچک و نیاز به دقت بالا
- ❌ عدم وجود کلیدهای میانبر (مثل Enter برای رفتن به فیلد بعدی)

#### 5. **تجربه کاربری**
- ❌ در موبایل نیاز به اسکرول افقی برای دیدن همه فیلدها
- ❌ فاصله‌گذاری بین المان‌ها کم است
- ❌ دکمه حذف کوچک و در گوشه قرار دارد

### 2. طراحی Responsive (اولویت: بالا)

#### ساختار پیشنهادی:

**دسکتاپ (≥1240px):**
- استفاده از `Row` با `Expanded` برای چیدمان افقی
- نمایش همه فیلدها در یک سطر (در صورت امکان)
- استفاده از `DataTable` یا ساختار مشابه برای نمایش منظم

**تبلت (600-1240px):**
- ترکیب `Row` و `Column` برای فیلدهای مهم
- فیلدهای کم‌اهمیت در سطر دوم

**موبایل (<600px):**
- تبدیل کامل به `Column` برای هر ردیف
- هر فیلد در یک سطر کامل
- استفاده از `Card` برای جداسازی بصری ردیف‌ها

#### کد نمونه:
```dart
Widget _buildRow(BuildContext context, int index, InvoiceLineItem item) {
  final isMobile = ResponsiveHelper.isMobile(context);
  final isTablet = ResponsiveHelper.isTablet(context);
  final fieldHeight = ResponsiveHelper.responsiveValue(
    context,
    mobile: 56.0,
    tablet: 52.0,
    desktop: 48.0,
  );

  if (isMobile) {
    return _buildMobileRow(context, index, item, fieldHeight);
  } else if (isTablet) {
    return _buildTabletRow(context, index, item, fieldHeight);
  } else {
    return _buildDesktopRow(context, index, item, fieldHeight);
  }
}
```

### 3. قابلیت مرتب‌سازی ردیف‌ها (اولویت: متوسط)

#### پیشنهادات:

**گزینه 1: Drag & Drop**
- استفاده از `ReorderableListView` یا `ReorderableWrap`
- نمایش آیکون drag handle (☰) در ابتدای هر ردیف
- امکان جابجایی با drag & drop

**گزینه 2: دکمه‌های جابجایی**
- دکمه‌های ↑ و ↓ در کنار شماره ردیف
- برای کاربرانی که drag & drop را ترجیح نمی‌دهند

**گزینه 3: ترکیبی**
- در دسکتاپ: drag & drop
- در موبایل: دکمه‌های ↑↓ (بهتر برای لمس)

#### کد نمونه:
```dart
// استفاده از ReorderableListView
ReorderableListView(
  onReorder: (oldIndex, newIndex) {
    setState(() {
      if (oldIndex < newIndex) {
        newIndex -= 1;
      }
      final item = _rows.removeAt(oldIndex);
      _rows.insert(newIndex, item);
    });
    _notify();
  },
  children: _rows.asMap().entries.map((e) {
    return _buildRowWithDragHandle(context, e.key, e.value);
  }).toList(),
)
```

### 4. بهبود سرعت ثبت (اولویت: بالا)

#### پیشنهادات:

**الف) Navigation با کلیدهای صفحه‌کلید:**
- `Tab` یا `Enter`: رفتن به فیلد بعدی
- `Shift+Tab`: برگشت به فیلد قبلی
- `Ctrl+Enter` یا `Alt+Enter`: افزودن ردیف جدید
- `Delete`: حذف ردیف (وقتی فیلد خالی است)

**ب) Auto-focus:**
- بعد از انتخاب کالا، فوکوس خودکار به فیلد تعداد
- بعد از وارد کردن تعداد، فوکوس به قیمت واحد
- بعد از وارد کردن قیمت، فوکوس به شرح (یا ردیف بعدی)

**ج) پیش‌فرض‌های هوشمند:**
- تعداد پیش‌فرض: 1
- واحد پیش‌فرض: واحد اصلی کالا
- قیمت پیش‌فرض: قیمت پایه کالا (بر اساس نوع فاکتور)
- انبار پیش‌فرض: انبار پیش‌فرض کالا

**د) دکمه افزودن سریع:**
- دکمه `+` بزرگ و همیشه در دسترس
- در موبایل: `FloatingActionButton` در پایین
- در دسکتاپ: دکمه در انتهای جدول یا نوار ابزار
