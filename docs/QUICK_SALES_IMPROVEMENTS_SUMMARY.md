## 📝 موارد باقی‌مانده (پیشنهادات برای آینده)

### 1. اعتبارسنجی موجودی قبل از افزودن به سبد
**وضعیت**: بررسی موجودی در سرور انجام می‌شود و خطا برمی‌گرداند
**توصیه**: 
- برای UI می‌توان یک هشدار نمایش داد اما اجازه افزودن داد
- بررسی کامل موجودی در زمان ثبت فاکتور در سرور انجام می‌شود
- این رویکرد بهتر است چون:
  - سریع‌تر (بدون API call اضافی)
  - موجودی در زمان ثبت دقیق‌تر است
  - کاربر می‌تواند محصول را به سبد اضافه کند و بعد تصمیم بگیرد

### 2. نمایش موجودی در سبد خرید
**وضعیت**: نیاز به API call برای هر محصول
**توصیه**:
- می‌توان در آینده اضافه شود
- نیاز به endpoint جدید یا bulk API برای دریافت موجودی چند محصول
- ممکن است کند باشد برای سبدهای بزرگ

### 3. Debounce برای جستجو
**وضعیت**: در حال حاضر جستجو فقط با Enter یا دکمه انجام می‌شود
**توصیه**:
- برای جستجوی خودکار (live search) می‌توان اضافه شود
- در فروش سریع، جستجو با Enter مناسب‌تر است
- می‌توان به عنوان قابلیت اختیاری در تنظیمات اضافه شود

### 4. History محصولات اخیر
**وضعیت**: قابلیت جدید پیشنهادی
**توصیه**:
- می‌توان در localStorage یا backend ذخیره شود
- نمایش به صورت Quick Access در UI
