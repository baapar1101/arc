**راه‌حل اعمال شده:**

- حالا reference به صورت `$node_id.field_name` ذخیره می‌شود

**فایل تغییر یافته:**
- `hesabixUI/hesabix_ui/lib/widgets/workflow/workflow_node_config_dialog.dart`

**ویژگی‌های جدید:**

### 2️⃣ مشکل داده‌های ورک‌فلو در دیتابیس (حل شد ✅)

**مشکل:**
- ورک‌فلو شماره 3 دارای کانفیگ نادرست بود
- فیلد `message` به جای متن، reference نادرست `$node_id` داشت
- پیامی به تلگرام ارسال نمی‌شد

**راه‌حل اعمال شده:**
- داده‌های ورک‌فلو در دیتابیس اصلاح شد
- فیلد `message` به یک متن معنادار با reference های صحیح تغییر کرد

**قبل از اصلاح:**
```json
{
  "message": "$6a6f62bf-1680-4a50-9735-a1acc293d0d0"
}
```

**بعد از اصلاح:**
```json
{
  "message": "🔔 فاکتور جدیدی ایجاد شد!\n\n📋 کد فاکتور: $6a6f62bf-1680-4a50-9735-a1acc293d0d0.invoice_code\n💰 مبلغ کل: $6a6f62bf-1680-4a50-9735-a1acc293d0d0.total_amount\n👤 مشتری: $6a6f62bf-1680-4a50-9735-a1acc293d0d0.customer_name"
}
```

## 📋 فیلدهای پیشنهادی برای انواع نودها

Reference Selector حالا بر اساس نوع نود، فیلدهای مناسب را پیشنهاد می‌دهد:

### فاکتورها (invoice):
- `invoice_id` - شناسه فاکتور
- `invoice_code` - کد فاکتور
- `invoice_number` - شماره فاکتور
- `total_amount` - مبلغ کل
- `customer_name` - نام مشتری
- `status` - وضعیت
- و موارد دیگر...

### پرداخت‌ها (payment):
- `payment_id` - شناسه پرداخت
- `amount` - مبلغ
- `payment_method` - روش پرداخت
- `reference_code` - کد پیگیری
- و موارد دیگر...

### مشتریان (person/customer):
- `person_id` - شناسه
- `name` - نام
- `email` - ایمیل
- `phone` - تلفن
- و موارد دیگر...

## 🚀 نحوه استفاده از Reference Selector جدید

### مرحله 1: کلیک روی دکمه "انتخاب از نودهای قبلی"

<img src="docs/workflow_reference_step1.png" alt="مرحله 1" />

### مرحله 2: انتخاب نود مورد نظر

لیست تمام نودهای قبلی نمایش داده می‌شود. نود مورد نظر را انتخاب کنید.

### مرحله 3: انتخاب فیلد خاص

دو گزینه دارید:
1. **استفاده از کل خروجی نود** (برای موارد خاص)
2. **انتخاب یک فیلد خاص** از لیست فیلدهای پیشنهادی

### نتیجه:

Reference به صورت `$node_id.field_name` در فیلد ذخیره می‌شود.
