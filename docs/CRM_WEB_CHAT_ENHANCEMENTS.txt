CRM web chat — planned/follow-up (device, IP, read ticks, history)
================================================================

A) Analysis summary
--------------------
1) Device (mobile / tablet / desktop): optional field `device_type` in POST
   /api/v1/public/crm-chat/conversations/start; stored in
   `crm_chat_conversations.extra_metadata.device_type` (set by client).
   Server also stores `extra_metadata.visitor_ip` from `get_client_ip(request)`.
2) Operator UI (Hesabix panel): read `extra_metadata` on each conversation;
   show small device icon + tooltip, and "IP: …" for support.
3) Read receipts: visitor messages already have `read_at` when the agent
   marks read; the WordPress widget already draws double-tick SVG and
   `applyMessagesReadFromEvent` should only update *visitor* rows when
   `reader_role === 'agent'` (so agent-side read receipts stay correct).
4) History: keep a local list of `{ conversation_id, visitor_token }` in
   localStorage (per apiBase+publicKey), push the current session before
   "new chat" or before starting a second conversation; add a <select> in
   the chat step to switch threads and reload messages + WebSocket.
