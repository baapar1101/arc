### کدهای اضافه شده

#### متدهای جدید:

1. **`_loadProductStock(int productId)`**
   - بارگذاری موجودی یک محصول
   - فقط اگر انبار انتخاب شده باشد
   - جلوگیری از درخواست‌های تکراری

2. **`_refreshAllStocks()`**
   - به‌روزرسانی موجودی تمام محصولات سبد
   - Bulk request برای بهینه‌سازی
   - فقط برای محصولات با `trackInventory = true`

3. **`_getProductStock(int? productId)`**
   - دریافت موجودی از Map
   - Helper method ساده

4. **`_hasInsufficientStock(InvoiceLineItem item)`**
   - بررسی موجودی ناکافی
   - برای نمایش هشدار استفاده می‌شود

#### State Variables جدید:

```dart
Map<int, num> _productStocks = {};  // موجودی محصولات
bool _loadingStocks = false;         // وضعیت لودینگ
```

### جریان کار (Workflow)

1. **افزودن محصول به سبد:**
   - اگر `trackInventory = true` → موجودی لود می‌شود
   - موجودی در UI نمایش داده می‌شود

2. **افزایش تعداد:**
   - بررسی موجودی قبل از افزایش
   - جلوگیری در صورت نبود موجودی کافی

3. **حذف محصول:**
   - حذف موجودی از Map اگر آخرین مورد است

4. **ثبت فاکتور:**
   - پاک کردن موجودی‌ها
   - به‌روزرسانی موجودی برای استفاده بعدی

### بهبودهای Performance

1. **Lazy Loading**: موجودی فقط وقتی نیاز است لود می‌شود
2. **Bulk Requests**: یک درخواست برای همه محصولات
3. **Caching**: ذخیره در Map برای جلوگیری از درخواست‌های تکراری
4. **Conditional Loading**: فقط برای محصولات با `trackInventory = true`

### پیام‌های خطا و هشدار

1. **موجودی ناکافی در افزایش تعداد:**
   - "موجودی کافی نیست. موجودی: X"

2. **هشدار در کارت محصول:**
   - "موجودی کافی نیست! موجودی: X، درخواست: Y"

3. **در حال بررسی موجودی:**
   - "در حال بررسی موجودی..." (تا زمانی که لود شود)

### نکات مهم

1. **کالاهای یونیک**: موجودی برای کالاهای یونیک نمایش داده نمی‌شود (چون instance_id دارند)

2. **بدون انبار**: اگر انبار پیش‌فرض تنظیم نشده باشد، موجودی لود نمی‌شود

3. **Error Handling**: در صورت خطا در دریافت موجودی، فقط لاگ می‌شود و UI ادامه می‌دهد

4. **Async Operations**: تمام عملیات async به درستی مدیریت می‌شوند (mounted checks)
