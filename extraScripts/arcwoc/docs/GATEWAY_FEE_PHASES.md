# کارمزد درگاه و Fee ووکامرس — فازهای اجرایی (ArcWOC)

## فاز ۱ — Fee سفارش (مشتری)

**هدف:** کارمزدهایی که روی سفارش ووکامرس به مشتری اضافه می‌شود (مثل «کارمزد خرید اقساطی»).

| حالت | تنظیم `fee_line_mode` | حسابداری در حسابیکس |
|------|------------------------|------------------------|
| خط خدمت (پیش‌فرض) | `service` | درآمد در حساب 50001 (فروش کالا) |
| اضافات فاکتور | `account_adjustment` | بستانکار 60101 (یا حساب انتخابی) |

Fee منفی:

| حالت | `fee_negative_mode` |
|------|---------------------|
| تخفیف خط | `line_discount` (پیش‌فرض) |
| کسورات فاکتور | `deduction_adjustment` → بدهکار 70902 |

گزینه `fee_exclude_from_profit`: کارمزد در گزارش سود فاکتور به‌عنوان سود کالا محاسبه نمی‌شود.

## فاز ۲ — کارمزد تسویه درگاه (پذیرنده)

**هدف:** مبلغی که درگاه از واریز کسر می‌کند؛ **بدون تغییر** مبلغ فاکتور/بدهی مشتری.

| حالت | `gateway_settlement_mode` |
|------|---------------------------|
| غیرفعال | `off` |
| درصد از مبلغ سفارش | `percent` |
| مبلغ ثابت | `fixed` |
| قواعد per روش پرداخت | `rules` — هر خط: `payment_method\|درصد\|ثابت` |
| متای سفارش | `order_meta` |

در سند **دریافت** همراه فاکتور: فیلد `commission` روی خط بانک/صندوق → ثبت خودکار در 70902.

فیلتر توسعه: `hesabix_v2_order_gateway_commission`

## فاز ۳ — یکپارچه‌سازی درگاه‌ها (اختیاری)

- افزونه درگاه ووکامرس متای کارمزد واقعی را در `_gateway_settlement_fee` یا کلید سفارشی بنویسد.
- یا از `hesabix_v2_order_gateway_commission` برای محاسبه اختصاصی استفاده شود.

## فاز ۴ — پنل حسابیکس (آینده)

نمایش/ویرایش همین کلیدهای `sync_settings` از تب تنظیمات ووکامرس در UI حسابیکس (پل REST).

## پیش‌فرض توصیه‌شده برای فروش اقساطی

```
fee_line_mode = account_adjustment
fee_exclude_from_profit = true
gateway_settlement_mode = rules
# digipay|2.5|0
```

## پیش‌فرض برای درگاه واسط با متای کارمزد

```
gateway_settlement_mode = order_meta
gateway_settlement_meta_key = _gateway_settlement_fee
```
